# Changelog

All notable changes to this project will be documented in this file.

## [v1.1.1] - 2026-06-26

### Fixed
- **Cache Removal**: Khắc phục triệt để lỗi sử dụng lại bản dịch cũ (caching) mặc dù người dùng đã yêu cầu dịch mới mỗi lần. Xóa bỏ hoàn toàn hệ thống lưu trữ cache LRU ngầm (`_cache_put`, `cache_order`, `self.cache`) và check cache trong module biên dịch (`translator.py`) cũng như các default options liên quan trong `config.py`. Điều này đảm bảo mỗi request dịch thuật luôn là fresh-request tới nhà cung cấp API.

## [v1.0.9] - 2026-06-26

### Added
- **Smart Model Filtering v2.0**: Đã triển khai thuật toán "kẹp đôi" (Whitelist + Blacklist + Regex) siêu cấp khi load model API:
  - Chỉ cho phép các LLM mạnh vượt qua (như gemini, gemma, llama, mistral, qwen...).
  - Quét sạch các model chuyên biệt dư thừa (như embed, vision, image, tts, whisper, ocr, guard, math...).
  - Tự động rút gọn bằng cách lọc bỏ các phiên bản đuôi ngày tháng hoặc thử nghiệm rác rưởi (-preview, -exp, -001).
  - Tối ưu hóa cực độ: Gemini giảm từ 50 xuống 15 models, Mistral từ 77 xuống 20 models, Groq từ 17 xuống 5 models.
  - Bổ sung `gpt` vào Whitelist và đặt model mặc định cho GroqCloud là `gpt-oss-120b`.
  - Mở rộng Blacklist loại bỏ triệt để các model dư thừa của Mistral (`vibe`, `tiny`, `small`, `open`), chỉ giữ lại các model hạng nặng (`large`, `medium`).
  - Khắc phục triệt để lỗi từ khóa `open` của Mistral filter bóp nhầm model `openai/gpt-oss-120b` của Groq.
- **Groq API Update**: Đã chèn thêm `User-Agent` vào header khi gọi API của GroqCloud để xử lý và bypass hoàn toàn lỗi `HTTP Error 403: Forbidden`.
- **Live Sequential Testing**: Nâng cấp nút "Test". Từ nay, khi người dùng vừa load model xong và bấm Test, hệ thống sẽ gửi 1 request test dịch thuật **lần lượt** tới tất cả các model đang hiển thị trong Combobox. Những model nào báo lỗi (404, 403, 429...) sẽ bị gạch tên ngay lập tức, Combobox sẽ tự động được dọn dẹp và chỉ giữ lại 100% các model đang sống và hoạt động tốt.
- **UI & UX Refinements**: 
  - Nới rộng khoảng trống cho trạng thái "Testing..." để không bị đè chữ lên nút "Lưu Key".
  - Tự động làm mờ và vô hiệu hóa (disabled) Combobox model, nút "Load Models", nút "Lưu Key" và nút "Test" trong quá trình test tự động đang chạy.
  - Đổi link lấy API key của Mistral thành đường dẫn trực tiếp: `https://console.mistral.ai/api-keys`.

## [v0.4.6] - 2026-06-26

### Fixed
- **Global Hotkey Drop (Silent Disconnect)** (`hotkey_manager.py`): Replaced the `keyboard` library's hook-based global hotkeys with Windows Native `RegisterHotKey` via `ctypes`. This fixes a critical issue where Windows would silently drop the Low-Level Keyboard Hook (`WH_KEYBOARD_LL`) due to the `LowLevelHooksTimeout` limit when the system was busy, causing the app to stop responding to hotkeys after a period of time or heavy usage. `keyboard` is now only kept in `requirements.txt` for the temporary hotkey recording UI in the Settings dialog.

## [v0.4.5] - 2026-06-25

### Fixed
- **Hotkey Race Condition & DevTools Bug** (`clipboard_util.py`): Fixed an issue where pressing the translation hotkey extremely fast caused the app to fail copying text and erroneously open the browser's DevTools. The OS often interleaves physical KeyUps with synthetic `Ctrl+C` commands if the user releases keys rapidly. Resolved by unconditionally force-releasing all modifier states (both generic and specific left/right variants) before simulating `Ctrl+C`, guaranteeing a clean key state for the clipboard operation.

## [v0.4.4] - 2026-06-25

### Fixed
- **Silent Exception NameError in Translation Popup** (`main.py`): Fixed `NameError` crash where the error display in the popup window failed to render, leaving the popup stuck at "⏳ Translating..." (pending). The callback was referencing the exception variable `e` which was garbage-collected at the end of the `except` block. Fixed by freezing the error message value via early binding in lambda.
- **Copy Failure and Unintended Browser Behavior (F12 DevTools)** (`clipboard_util.py`): Fixed a critical bug where simulating `Ctrl+C` caused text copy to fail and sometimes unintentionally opened the browser's Developer Tools (F12) or focused the menu bar. Resolved by using `win32api.GetAsyncKeyState` to release specific Left/Right physical modifier keys (`VK_LSHIFT`, `VK_RSHIFT`, etc.) with proper `KEYEVENTF_EXTENDEDKEY` flags instead of generic keys.
- **Gemini API 503 / 429 Errors** (`translator.py`): Implemented a robust retry mechanism with exponential backoff (up to 3 attempts, starting at 1.0s delay) for both Gemini API and Google Free Translate to auto-recover from temporary service unavailability (503) or rate limits (429) before throwing errors.
- **Log Management** (`main.py`): Implemented `TimedRotatingFileHandler` to limit `directtrans.log` to retain information only for the last 1 hour, preventing infinite file size growth.

### Changed
- **Default Translation Window Size**: Expanded the default dimensions of the translation popup from `380x220` to `880x465` to improve readability and fit larger translated paragraphs without clipping.
- **Default Gemini Model**: Updated the default Gemini model from `gemini-1.5-flash` to `gemini-3.1-flash-lite` across configuration defaults (`config.py`), translation manager (`translator.py`), and default `config.json`.

## [v0.4.2] - 2026-06-25

### Fixed
- **Critical Fallback Popup Crash** (`main.py`): Fixed `AttributeError` crash when Replace mode fails and the app attempts to show a fallback popup. The `PopupWindow` constructor was missing the required `config` argument, causing an immediate crash when trying to read font/language settings.
- **Async TclError on Window Close** (`settings_window.py`): Added `winfo_exists()` guard to all 7 UI-update callbacks dispatched via `window.after(0, ...)` inside background threads (`fetch_models`, `_test_key`). Prevents `TclError: invalid command name` crash when the user closes Settings while an API operation is in progress.
- **MouseWheel Event Leak** (`settings_window.py`): Replaced `canvas.bind_all("<MouseWheel>")` with `canvas.bind("<MouseWheel>")`. The `bind_all` form was intercepting scroll events from child dialogs (e.g. the hotkey editor), causing the settings canvas to scroll when the user scrolled inside a sub-dialog.
- **Tray Icon Color Inconsistency** (`tray.py`): Updated the PIL-generated fallback tray icon color from `#89b4fa` (leftover Catppuccin Latte blue) to `#cc785c` (Warm Coral), consistent with the Claude Design palette used across the rest of the app.

### Changed
- **Settings Window Topmost Behavior** (`settings_window.py`): The settings window no longer stays permanently on top of all other applications. It briefly becomes topmost when opened (200ms) to attract focus, then reverts to normal z-order.
- **Tray Menu Localization** (`tray.py`): The system tray context menu (Settings, Quit, Enable/Disable) now respects the active UI language setting. Supports all 5 languages: Vietnamese, English, Korean, Chinese, Japanese.
- **Color Variable Naming** (`settings_window.py`): Renamed misleading color constants `BLUE`, `GREEN`, `MAUVE`, `PEACH` (all were `#cc785c`) to a single `PRIMARY` constant, matching the Claude Design spec. Eliminates developer confusion where `self.GREEN` visually appeared coral/warm-orange.
- **DPI Awareness** (`main.py`): Application now calls `SetProcessDpiAwareness(1)` at startup (Windows 8.1+) before creating the Tk root window, enabling crisp, correctly-scaled rendering on 4K and 125%/150% scaled displays.
- **Popup Event Timeout Handling** (`main.py`): When the UI thread fails to create a translation popup within the 2-second timeout window, the translation result is now explicitly discarded with a warning log instead of silently dropped.
- **Replace Paste Delay** (`popup_window.py`): Increased the hardcoded focus-restoration delay before paste simulation from 150ms to 250ms for better reliability on slower machines. Added explanatory comment.
- **Font Initialization Safety** (`settings_window.py`): Wrapped the `update_ui_language()` call in `__init__` with a try/except so a font-mapping failure does not prevent the settings window from opening.
- **Removed Dead Hover Binding** (`settings_window.py`): Removed no-op `<Enter>`/`<Leave>` bindings on the Gemini API key link label that set the same color in both directions, producing no visible effect.
